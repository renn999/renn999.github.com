Use SSH Open a SOCKS Proxy

在此 感謝Billcho的教導

從來都沒想過SSH的功能會這麼的強大 連開一個臨時Proxy都可以

基本上在Linux中去找ssh的男人(men)時 會發現到一個參數

-D [bind_address:]port
Specifies a local dynamic'' application-level port forwarding. This works by allocating a socket to listen to port on the local side, optionally bound to the specified bind_address. Whenever a connection is made to this port, the connection is forwarded over the secure channel, and the application protocol is then used to determine where to connect to from the remote machine. Currently the SOCKS4 and SOCKS5 protocols are supported, and ssh will act as a SOCKS server. Only root can forward privileged ports. Dynamic port forwardings can also be specified in the configura-tion file.<br /> IPv6 addresses can be specified with an alternative syntax: [bind_address/]port or by enclosing the address in square brackets. Only the superuser can forward privileged ports. By default, the local port is bound in accordance with the GatewayPorts setting. However, an explicit bind_address may be used to bind the connection to a specific address. The bind_address oflocalhost'' indicates that the listening port be bound for local use only, while an empty address or '*' indicates that the port should be available from all interfaces.

基本上來說 這個參數就是利用SSH在 遠端的電腦上開一個Port之後 來當作一個Socks Proxy

Linux的作法老實說比較簡單 因為Linux不可能不具備SSH吧...XD

所以說的話只要SSH到目的主機參數多加上-D <你要開的Prot>這樣就OK了 連線方式就是在Socks5 proxy設定那邊打上127.0.0.1:<你要開的Prot> 這樣就好了 可以連去測試IP的地方看一看

如果你有一個工作Host端伺服器為Linux 但是你的client為Windows的話 或者你黑進去一個Linux的伺服器打算要在上面開一個Port的話 這個部份就要注意一點了 你比需要在Host端開一個port 如果直接下-D <你要開的Prot>的話這樣是沒有辦法連上去的 因為他裡面有說 如果沒有指定bind_address的話他只會允許localhost的連線 故要用-D <Host端IP>:<你要開的Prot> 這樣才可以連的上去
